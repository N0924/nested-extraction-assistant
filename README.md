# 嵌套解压助手

Windows 本地桌面工具：从一个根文件开始，使用 WinRAR 识别并递归解压普通压缩包、错误扩展名文件、MP4/TXT 等外壳中的嵌入式 ZIP、Apate 伪装文件和常见分卷。

## 当前能力

- 所有源文件只读，不改名、不覆盖、不移动、不删除；
- WinRAR 在后台测试和解压 ZIP、RAR、7Z 等其自身支持的格式；
- 文件扩展名不可信，ZIP/RAR/7Z 即使被直接改成 MP4、TXT 等后缀也会先交给 WinRAR 按内容判断；若 WinRAR 受错误后缀影响，程序只在正式文件头匹配时生成一份正确后缀的工作副本重试；
- 对带有真实 MP4/TXT 前置外壳的 ZIP，程序会从 ZIP 目录结构反推出边界，只把压缩包区间流式复制到独立工作副本；
- 直接格式、正确后缀副本和嵌入式 ZIP 都不匹配时，再检查是否符合 Apate 归档特征并生成独立还原副本；
- 自动继续处理内层压缩包，不设置固定嵌套层数；
- 每个内层压缩包按“任务密码、密码池、无密码”的顺序实际解压；候选返回成功码但没有产生文件时继续尝试下一个；
- 格式路线由程序自动按“直接识别、正确后缀副本、嵌入式 ZIP、Apate”顺序验证，不要求用户预先判断伪装类型或选择解压器；
- 识别 `0zip/1zip/2zip`、`partN.rar`、`name.7z.001`、旧式 RAR 分卷和标准 ZIP 分卷；
- `name.7z.001/.002/.003` 这类原始字节切分卷会在工作目录流式拼接为临时完整归档，再交给 WinRAR；
- 当同目录只有一个同名候选首卷时，`name.mp4 + name.002 + name.003`、`name.txt + name.002 + name.003` 也会自动按首卷和连续编号归组、拼接；
- 最终文件按任务隔离到“输出目录”；
- 多文件选择或直接拖入文件、文件夹，每个根文件建立一条独立任务；
- 用表格查看等待、运行、当前处理阶段、结果目录和历史状态；
- 双击任务的密码栏或点击“编辑密码”，直接为该任务填写优先密码；
- 设置菜单提供长期密码池，第一项为默认密码，可按换行或 `|` 批量添加；
- 任务表支持多选、右键开始当前任务或所选任务，也可继续开始全部等待任务；
- 同时运行任务默认为 1 个，可设置为 1–5 个；文件数、累计输出容量和压缩包数量默认不限制，也可分别启用自定义上限；
- 每次运行都在 `runtime/logs/<任务标识>/` 追加可读的 `task.log` 和便于程序分析的 `task-events.jsonl`，可从任务表直接打开；
- 任务历史和设置会保存在本机；任务密码不保存，长期密码池按用户选择以明文 JSON 保存在本机，并在密码池窗口中直接显示。

## 运行条件

- Windows；
- Python 3.11 或更高版本；
- 已安装 WinRAR。本机已验证 WinRAR 7.0.0；
- 不需要安装 Python 第三方包。

## 运行

在本目录打开 PowerShell：

```powershell
python .\app.py
```

在窗口中操作：

1. 从“文件”菜单或工具栏添加一个或多个根文件或文件夹，也可直接拖进任务表；
2. 在“设置 → 密码池”添加常用密码，第一项作为默认密码；
3. 某个文件需要特殊密码时，双击该任务的密码栏直接填写；留空则使用密码池；
4. 从“设置”菜单配置工作目录、输出目录、`WinRAR.exe`、可选任务上限和同时运行数；
5. 多选任务后点击“开始所选”，或右键开始当前任务；也可点击“开始全部”；
6. 完成后选择任务并打开结果目录。

工作目录和输出目录必须彼此独立，也不能放在源文件所在目录内。任务完成后可直接从界面打开结果目录。

拖入文件夹时只查看该文件夹第一层，不递归扫描子文件夹。对于 `452.7z.001 / .002 / .003` 这类分卷，可以拖入整个文件夹，也可以只拖任意一卷；程序会归组并只建立 `.001` 任务。后续分卷必须留在同一文件夹中，文件名不能改变。

对于 `bundle.mp4 + bundle.002 + bundle.003` 这类首卷被改名的分卷，也可以拖入文件夹或任意一卷。只有同目录中存在唯一的同名首卷候选、并且后续编号从 `.002` 连续排列时，程序才会自动归组；有多个候选或缺卷时不会猜测。

扩展编号分卷的拼接副本会额外占用约等于全部压缩分卷总和的工作磁盘空间。例如三卷合计 1.1 GB，工作目录至少还需要约 1.1 GB 才能开始解压。源分卷始终只读。

嵌入式 ZIP 也会在工作目录生成一份去除 MP4/TXT 外壳的临时副本，占用空间约等于其中压缩包本身的大小。它不依赖外层文件扩展名；源文件不会被改名、截断或覆盖。

文件头明确是 ZIP/RAR/7Z、但必须使用正确后缀才能被 WinRAR 读取时，程序会额外生成一份与源文件等大的临时副本。普通视频、图片和无法确认格式的文件不会触发这次整文件复制。

表格中的进度是当前处理阶段，例如“测试归档”或“正在处理第 2 层”，不是估算百分比。嵌套层数和解压后文件量事先未知，因此当前版本不显示可能误导用户的百分比。

源码运行时，设置、历史、日志和密码池保存在项目的 `runtime/`；打包版保存在 `%LOCALAPPDATA%\NestedExtractionAssistant\`。工具栏“任务日志”直接打开全部任务汇总，右键任务可打开单任务日志。长期密码池以明文保存，方便用户直接核对；旧版加密或 Windows 凭据格式需要在密码池窗口中重新保存一次。程序意外关闭后，原先等待或运行的任务会显示为“已中断”，但不会自动继续；需要从根文件重试。

## 安全边界

- 程序不调用 `7z.exe` 或 `apate.exe`；Apate 逻辑是根据公开文件布局重新实现；
- WinRAR 使用 `t` 和 `x` 命令读取源文件，所有解压输出写入独立任务目录；
- 嵌入式 ZIP 只有在结束目录、中央目录和本地文件头能够互相校验时才会生成副本，零散出现的 `PK` 字节不会触发；
- 解压时禁用覆盖并跳过链接，完成后再次检查所有输出路径；
- 解压出的文件不会被执行；
- 单任务优先密码只保留在当前界面内存中，任务结束后清除；
- 常用密码池最多 50 个，直接显示并以明文 JSON 长期保存；能够读取该本地文件的人也能看到这些密码；
- 密码池窗口、任务密码栏、运行进度和任务日志均明文显示实际密码；任务历史、失败说明和结果目录不额外保存密码；
- WinRAR 7.0.0 的无人值守调用需要通过命令行参数接收密码，因此运行期间密码会短暂出现在本机进程参数中；
- 三项资源上限默认不限制；用户启用自定义上限后，程序在每一层解压完成后核对。无论是否设置上限，创建工作副本前仍会检查磁盘剩余空间；
- WinRAR 即使对某个候选返回成功码，只要该候选的独立输出目录完全为空，程序就会清理空目录并继续尝试下一个候选；
- 所有候选都没有产生文件时，任务才会要求补充密码；重新运行仍会从根文件开始，不支持断点续作；
- 成功任务会在结果移动完成并核对后删除可重新生成的工作副本；失败任务只保留不含密码的 `failure.json`。旧版本已经产生的工作目录不会被自动删除。
- 任务日志会记录密码、私人文件名、源路径和输出路径；`runtime/` 默认被 Git 忽略，公开项目前不得提交真实日志。

自定义连续数字文件会按分卷组处理。如果同一目录中的 `video0.zip`、`video1.zip` 实际是互不相关的独立压缩包，应分别放入不同目录或逐个处理。

## 程序图标与打包

程序图标位于 `assets/app.ico`，内含 16、24、32、48、64、128 和 256 像素七种 Windows 图标尺寸。源码运行时，主窗口、任务栏和“关于”窗口会自动使用该图标；`assets/app-logo.png` 用于“关于”窗口中的图案。本次没有修改主界面背景。

安装开发依赖后执行发布脚本：

```powershell
python -m pip install -r .\requirements-dev.txt
powershell -ExecutionPolicy Bypass -File .\packaging\build_release.ps1
```

脚本会在 `release/` 生成 Windows 便携版 ZIP 和 SHA-256 校验文件。ZIP 内的主程序名为 `嵌套解压助手.exe`，同时包含运行库、说明和许可证。使用时必须完整解压 ZIP，不能单独移动 EXE。程序的资源定位同时兼容源码目录和 PyInstaller 打包目录。
