# 贡献规范

## 代码风格

- 使用 Python 3.11 或更高版本，四个空格缩进，单行不超过 100 个字符；
- 模块、函数和变量使用 `snake_case`，类使用 `PascalCase`，常量使用 `UPPER_CASE`；
- 新增或修改的函数应补充类型标注；模块和非直观的公共接口使用简短中文文档字符串；
- 注释只说明安全边界、设计原因、第三方程序特殊行为或不直观算法，不逐行复述代码；
- 不提交临时调试输出、未说明的兼容分支或与当前功能无关的重构。

## 隐私与安全

- 不提交真实密码、私人文件名、本机绝对路径、NAS 地址、任务日志、解压结果或真实样本；
- `runtime/`、`build/`、`dist/`、`release/` 和本机教学文件必须保持在 Git 忽略列表中；
- 文件处理改动必须保留源文件只读、输出隔离、防止覆盖和路径逃逸检查；
- 不自动执行解压得到的程序、脚本、快捷方式或安装文件。

## 提交前检查

在项目根目录执行：

```powershell
python -m compileall -q .
python -m unittest discover -v
```

涉及界面或 WinRAR 调用时，还需在 Windows 上进行对应的人工验收。
